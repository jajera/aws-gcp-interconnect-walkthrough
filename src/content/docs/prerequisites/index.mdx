Before diving in, make sure you have the right accounts, tools, and credentials. This section also covers cost expectations so there are no surprises.

<Aside type="tip">
  This walkthrough targets the **
  <Tooltip term="aws" /> ↔ <Tooltip term="gcp" label="Google Cloud" />
  ** path — the only GA partner for [AWS Interconnect –
  multicloud](https://aws.amazon.com/interconnect/) as of this writing (April 2026). Azure and OCI
  support is announced but not yet available.
</Aside>

## Accounts and permissions

You need an <Tooltip term="aws" /> account with permissions for Direct Connect, EC2, <Tooltip term="vpc" />, <Tooltip term="iam" />, and <Tooltip term="interconnect" label="AWS Interconnect – multicloud" />.

You also need a <Tooltip term="gcp" /> project with billing enabled and these APIs enabled:

- Compute Engine API
- Network Connectivity API (`networkconnectivity.googleapis.com`)

```bash
gcloud services enable compute.googleapis.com networkconnectivity.googleapis.com \
  --project=YOUR_GCP_PROJECT_ID
```

## Authentication

```bash
# AWS
aws sts get-caller-identity

# GCP — Terraform uses Application Default Credentials (ADC)
gcloud auth application-default login
gcloud config set project YOUR_GCP_PROJECT_ID
gcloud auth application-default set-quota-project YOUR_GCP_PROJECT_ID
```

<Tooltip term="terraform" /> and the Google provider read credentials from <Tooltip term="adc" />,
not from `gcloud config`. After `gcloud config set project`, align the ADC quota project to the same
GCP project — otherwise you may see a mismatch warning and hit unexpected quota/billing issues:

```text
WARNING: Your active project does not match the quota project in your local
Application Default Credentials file.
```

Run `set-quota-project` whenever you switch GCP projects for this demo.

## Region pairing

<Tooltip term="interconnect" label="Interconnect" /> is region-to-region. Pick a supported pair
before editing `terraform.tfvars`.

<Aside type="tip">
  Authoritative list: [AWS Interconnect regional
  availability](https://docs.aws.amazon.com/interconnect/latest/userguide/region-availability.html)
  (GCP mirror: [paired
  locations](https://cloud.google.com/network-connectivity/docs/interconnect/how-to/partner-cci-for-aws/paired-locations)).
</Aside>

<Aside type="caution">
  `ap-southeast-2` (Sydney) is **not** a supported Interconnect pair. The nearest APAC option is
  `ap-southeast-1` ↔ `asia-southeast1` (Singapore).
</Aside>
