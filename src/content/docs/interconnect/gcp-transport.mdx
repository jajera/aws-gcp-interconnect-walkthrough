## Step 4 — Create the GCP transport

Run from Cloud Shell or a machine with `gcloud` authenticated to your <Tooltip term="gcp" /> project.

### Set environment variables

```bash
export GCP_PROJECT="$(terraform output -raw gcp_project_id)"
export GCP_REGION="$(terraform output -raw gcp_region)"
export GCP_NETWORK="$(terraform output -raw gcp_vpc_network_name)"
export TRANSPORT_NAME="$(terraform output -raw gcp_transport_name)"
export ADVERTISED_ROUTES="$(terraform output -raw gcp_advertised_routes_for_transport)"
export ACTIVATION_KEY="paste-activation-key-from-step-3"

gcloud config set project "$GCP_PROJECT"
```

### Create transport

```bash
gcloud beta network-connectivity transports create "$TRANSPORT_NAME" \
  --region="$GCP_REGION" \
  --network="$GCP_NETWORK" \
  --advertised-routes="$ADVERTISED_ROUTES" \
  --activation-key="$ACTIVATION_KEY"
```

<Aside type="caution">
  When using `--activation-key`, bandwidth is embedded in the key — do **not** pass `--bandwidth`
  separately.
</Aside>

<Aside type="note">
  The command blocks while the long-running operation completes (`Waiting for operation ...`). That
  is normal — do not interrupt unless you intend to cancel the create.
</Aside>

### Check status

```bash
gcloud beta network-connectivity transports describe "$TRANSPORT_NAME" \
  --region="$GCP_REGION"
```

Full cross-cloud readiness is confirmed when <Tooltip term="interconnect" label="AWS interconnect" /> is `Available` and incoming peering routes appear (Step 6).
