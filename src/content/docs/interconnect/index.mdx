This is where the magic happens. You'll create the cross-cloud link that connects your AWS and GCP networks over a managed, private path.

## The three manual steps

These steps can't be automated in Terraform yet. Each builds on the previous:

1. **[AWS Interconnect (console)](/aws-gcp-interconnect-walkthrough/interconnect/aws-interconnect/)** — Create the multicloud interconnect and get an activation key
2. **[GCP Transport (gcloud)](/aws-gcp-interconnect-walkthrough/interconnect/gcp-transport/)** — Use the activation key to create a Partner Cross-Cloud transport
3. **[VPC Peering (gcloud)](/aws-gcp-interconnect-walkthrough/interconnect/vpc-peering/)** — Peer your workload VPC to the transport's managed peering network

## How it flows

```text
AWS Interconnect (console)
       │
       ▼ activation key
GCP Transport (gcloud beta)
       │
       ▼ peeringNetwork
VPC Peering (gcloud)
       │
       ▼ routes flow
Private connectivity ✓
```

After all three are complete, AWS will transition from `pending` to `available` (typically 5–30 minutes). Then routes exchange and you can ping across clouds.
